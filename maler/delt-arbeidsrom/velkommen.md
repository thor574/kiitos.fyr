# Velkommen — ARBEIDSROMNAVN

Dette er den operative inngangen for arbeidsrommet `ARBEIDSROMNAVN`. Dette er et delt arbeidsrom.

- Universell kiitos-introduksjon: se fyret → `guide/kiitos.md`
- Arbeidsromsspesifikk orientering: [guide/kiitos.md](guide/kiitos.md)
- Begreper og navnekonvensjoner: se fyret → `styringsgrunnlag/begrepsmatrise.md`

## Kjerneinstruks

- Arbeidsrommet er en flerprosjekt-benk, ikke én prosjektrot.
- Svar og dokumentasjon skal som hovedregel være på norsk (bokmål) når det passer.
- Denne `velkommen.md` er kanonisk oppstartsflate for arbeidsrommet. Verktøyspesifikke portaler som `AGENTS.md`, `.github/copilot-instructions.md` eller brukerprofiler i andre KI-verktøy er tynne innganger som peker hit.
- Hvis en ny bruker mangler `kiitos.fyr`, `kiitos.ruff.*` eller relevante prosjektmapper i arbeidsrommet, hjelp først med å få disse lagt til før vanlig arbeid starter.
- Prosjektets `.kiitos/prosjektinstruks.md` er primær inngang for prosjektnær styring.
- Hold deg til aktiv sak. Parker sidefunn i `oppgaver/`.
- Ikke bryt API- eller databasekontrakter uten avklaring.
- Når stabile ord, visjoner, mål eller arkitekturvalg avklares, løft til laveste riktige styringsfil.
- Plassering av innhold, personvern og journal-lag: se fyret → `guide/plassering.md`.

## Leserekkefølge

1. `kiitos.fyr/velkommen.md` (universell orientering, via valgt oppstartsinstruks eller portalfil)
2. `kiitos.ruff.*/velkommen.md` (personlige preferanser, hvis den finnes)
3. Denne filen (arbeidsromsspesifikk orientering)
4. [guide/kiitos.md](guide/kiitos.md) for prosjektkart
5. Prosjektets `.kiitos/prosjektinstruks.md` når den finnes

## Arbeidsromgrenser

- Behandle dette arbeidsrommet som selvstendig.
- Universelle kiitos-guider, begreper og styringsgrunnlag eies av fyret.
- Når bruker sier `kiitos`, menes det samlede aktive instruksgrunnlaget, ikke én enkelt mappe.

## Arbeidsrommets aktive flater

- `guide/` — arbeidsromsspesifikke guider
- `styringsgrunnlag/` — stabile tverrprosjektlige føringer og valg
- `journal/` — tverrgående journal for arbeidsrommet
- `oppgaver/` og `spesifikasjoner/` — parkerte oppgaver og prosjekt-/systemkunnskap
- `analyse/` — analysenotater og utredninger
